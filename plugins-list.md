# Plugins

All plugins available in this marketplace.

| Plugin | Description |
| --- | --- |
| [fumadocs-engineer](./plugins/fumadocs-engineer) | Senior engineer for Fumadocs, the Next.js documentation framework — scaffolds, configures, and debugs docs sites and authors MDX content |
| [effect-ts](./plugins/effect-ts) | Senior engineer for Effect (Effect-TS), the typed functional TypeScript framework — services and layers, typed errors, Schema, concurrency, streams, testing, SQL, and ecosystem integrations |
| [zensical-writer](./plugins/zensical-writer) | Senior technical writer and engineer for Zensical, the static site generator by the Material for MkDocs team — scaffolds, configures, and authors documentation sites |
| [opencode-agent-cc](./plugins/opencode-agent-cc) | Run the headless OpenCode CLI from Claude Code for code review, adversarial review, second opinions, and write-capable rescue/fix tasks — respecting the user's configured provider and model |
| [pi-agent-cc](./plugins/pi-agent-cc) | Run the headless pi CLI (`pi -p`) from Claude Code for code review, adversarial review, second opinions, and write-capable rescue/fix tasks — respecting the user's configured provider and model |
| [lazy-senior-dev](./plugins/lazy-senior-dev) | Lazy senior dev mode. Forces the simplest, shortest solution that actually works: YAGNI, stdlib first, no unrequested abstractions. Activates on the /lazy-senior-dev:* commands or when its description matches — no always-on session hooks |
| [agent-host-protocol](./plugins/agent-host-protocol) | Reference for the Agent Host Protocol (AHP) — the JSON-RPC, channel-based state-sync protocol for multi-client AI agent sessions — covering channels, actions, reducers, write-ahead reconciliation, tool calls, and authentication |
